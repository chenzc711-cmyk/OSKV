# 仓库管理系统

这是一个仿参考图制作的仓库管理系统，包含左侧导航、仪表盘、库存查询、入库登记、出库登记、成本登记、商品管理和报表统计页面。

## 最简单使用方式

下载或复制单文件版本到电脑桌面：

```text
天猫仓库管理系统.html
```

然后直接双击打开即可使用，不需要 Bash、PowerShell、Python，也不需要安装依赖。数据会保存在当前浏览器本地。

> 如果你打开后仍然看到“天猫店铺 / 简洁清晰的店铺库存总览 / 红色按钮”的旧界面，说明你打开的是旧 ZIP 或浏览器缓存里的旧文件。请重新下载最新 ZIP，或直接打开本次新增/更新后的 `天猫仓库管理系统.html`。新版页面左侧是白色导航，并会显示“参考图新版”。



## 从 GitHub 下载 ZIP 时怎么判断对不对

你截图里点 **Code → Download ZIP** 的位置是正确的。下载前请确认页面文件列表里能看到：

```text
新版仓库管理系统.html
天猫仓库管理系统.html
index.html
app.js
styles.css
```

如果 GitHub 页面只看到 `天猫仓库管理系统.html`，或者 README 标题仍然是“天猫店铺仓库管理系统”，说明你正在看的还是旧页面/旧分支/旧缓存。请刷新网页，确认最新代码已经合并到 `main`，再重新点击 **Code → Download ZIP**。

下载并解压后，最推荐双击打开：

```text
新版仓库管理系统.html
```

打开后如果左侧看到“参考图新版”，就说明打开的是这次改过的新系统。


## 为什么 GitHub 首页看不到新版文件

如果仓库首页文件列表里还没有看到：

```text
新版仓库管理系统.html
天猫仓库管理系统.html
```

说明你当前看的 `main` 分支还没有合并最新修改。请先点页面顶部的 **“拉取请求 / Pull requests”**，打开最新的 PR，确认里面有这些新文件；等 PR 合并到 `main` 后，再回到 **代码 / Code** 页面下载。

注意：GitHub 网页冲突编辑器不支持处理 `.zip` 这类二进制文件。如果看到“不支持二进制文件”，不要在冲突页面打开 ZIP；直接删除/不要提交 ZIP 文件，合并后使用 GitHub 自带的 **Code → Download ZIP** 下载整个项目即可。

## 多电脑同步使用说明

当前版本是纯前端本地版，数据保存在每台电脑自己的浏览器里，因此不同电脑不会自动同步。如果需要多台电脑同步，有三种做法：

1. **临时共享**：在一台电脑导出 CSV，再到另一台电脑导入。
2. **局域网共享**：把系统放在一台固定电脑或 NAS 上，其他电脑通过同一个网址访问；但仍建议后续接数据库保存数据。
3. **正式多人同步**：增加后端、数据库和账号权限，把商品、入库、出库、成本数据统一保存到服务器。

## 本次视觉升级说明

新版界面升级为“星云仓储中枢”视觉风格：保留原来的左侧导航和页面结构，但强化了文字层级、矢量图标、玻璃拟态卡片、渐变指标、库存健康状态、智能预警说明、商品详情和报表 TOP 图，使每个模块更像完整的专业仓库管理后台。

## 功能

- 仪表盘：商品总数、总库存、库存总额、库存预警、今日/本月出入库、TOP 图表。
- 库存查询：搜索、库存状态筛选、库存预警、导出 CSV。
- 入库登记：新增入库、批量导入、导出 CSV、日期筛选。
- 出库登记：新增出库、批量导入、导出 CSV、日期筛选。
- 成本登记：新增成本、成本类型筛选、导出 CSV。
- 商品管理：新增、编辑、删除商品，搜索和导出。
- 报表统计：出入库统计、成本汇总、趋势图展示。

## PR 显示冲突时怎么办

如果 GitHub 的拉取请求页面提示“该分支存在必须解决的利益冲突”，说明这个 PR 是在旧代码基础上创建的，而 `main` 分支后来也改过同名文件。最简单的处理方式有两种：

### 方法一：直接在 GitHub 网页解决

1. 在 PR 页面点击 **解决冲突 / Resolve conflicts**。
2. 对冲突文件选择保留新版内容，也就是包含“参考图新版”“新版仓库管理系统.html”的版本；如果冲突文件是 `.zip`，不要在网页编辑它，直接不要提交 ZIP。
3. 删除 GitHub 自动插入的冲突标记：`<<<<<<<`、`=======`、`>>>>>>>`。
4. 点击 **标记为已解决 / Mark as resolved**。
5. 点击 **提交合并 / Commit merge**。
6. 回到 PR 页面，点击 **合并拉取请求 / Merge pull request**。

### 方法二：不会处理冲突时，用新的无冲突 PR

如果不熟悉冲突标记，建议关闭当前冲突 PR，然后从最新 `main` 重新创建一个分支，把新版文件重新提交到新分支，再开一个新的 PR。这样通常比手动改冲突更安全。

合并成功后，回到 **代码 / Code** 页面，文件列表里应该能看到：

```text
新版仓库管理系统.html
天猫仓库管理系统.html
```

如果需要 ZIP 包，直接在 **Code** 按钮里选择 **Download ZIP**，GitHub 会自动把这些文件打包下载。

### 你截图里的冲突编辑器怎么点

在 GitHub 冲突编辑器里，如果冲突块上方出现：

```text
Accept current change | Accept incoming change | Accept both changes
```

请按下面规则处理：

- 如果上半段或下半段里出现旧说明，例如“天猫店铺仓库管理系统”“Warehouse Control Center”“手动添加商品”，不要保留这一段。
- 如果其中一段出现新版说明，例如“参考图新版”“新版仓库管理系统.html”“仪表盘、库存查询、入库登记、出库登记、成本登记、商品管理、报表统计”，就保留这一段。
- 不要点 **Accept both changes**，否则新旧说明会同时留下，README 会重复且容易继续冲突。
- 如果你不确定，在这个 PR 里通常选择 **Accept incoming change** 更接近要合并进来的新版内容。

选完后确认页面里已经没有 GitHub 自动插入的冲突符号（也就是由多个小于号、等号、多个大于号组成的三行分隔符）。

然后点 **Mark as resolved / 标记为已解决**，最后点 **Commit merge / 提交合并**。


## “不支持二进制文件”错误说明

如果 GitHub 提示 **“不支持二进制文件”**，通常是因为你正在 PR 冲突页面里打开或处理 `.zip` 文件。ZIP 属于二进制文件，GitHub 网页不能像 README、HTML、CSS、JS 那样在线编辑或合并。

处理方式：

1. 不要在冲突页面处理 `.zip` 文件。
2. 保留并合并文本文件：`README.md`、`index.html`、`styles.css`、`app.js`、`新版仓库管理系统.html`、`天猫仓库管理系统.html`。
3. 合并成功后，回到仓库首页，点击 **Code → Download ZIP**，GitHub 会自动生成 ZIP 包。
4. 解压后双击 `新版仓库管理系统.html` 使用。

## 合并后文件看起来没更新怎么办

如果你已经合并 PR，但 GitHub 首页看起来还是旧说明，请按下面顺序确认：

1. 在文件列表中点击 `index.html`，按 `Ctrl + F` 搜索 `星云仓储中枢`。如果能搜到，说明新版代码已经合并，只是文件列表的提交说明看起来像旧的。
2. 点击 `app.js`，搜索 `tmallWarehouseV4`。如果能搜到，说明新版脚本已经合并。
3. 使用时不要打开以前下载到电脑里的旧文件，要重新点 GitHub 上的 `新版仓库管理系统.html`，进入文件页面后点 **Raw / 原始文件**，再按 `Ctrl + S` 保存到桌面。
4. 如果浏览器仍显示旧界面，按 `Ctrl + F5` 强制刷新，或者换一个浏览器打开新版 HTML。
5. 如果 `index.html` 里搜不到 `星云仓储中枢`，说明合并冲突时保留了旧内容，需要重新开一个 PR，把新版文件再次合并进去。
